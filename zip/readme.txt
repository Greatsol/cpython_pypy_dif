Для встроенного генератора zip() добавлен аргумент strict с значением False по умолчанию.
def zip(*iterables, strict=False)
При значении аргумента strict=True и разной длине передаваемых итерируемых объектов, в cpython3.10 будет вызвана ошибка ValueError с соответствующим указанием в отличии длины объектов.
А при strict=False количество элементов, возращаемых генератором, определяется по длине самого короткого объекта.
В pypy3.9 данного аргемента нет.

Команды для запуска
Cpython: python3.10 test.py
Pypy: pypy3.9 test.py
